# @TRC — Trace Relational Syntax

## Overview

Saves the SQL syntax generated by MRI statements into a report.

> **Prerequisite:** Before using `@TRC`, you must establish a database connection using the [`@LGN`](LGN.md) (Log On to Relational Database) statement.

The `@TRC` statement terminates when the run executes either a [`@LGF`](LGF.md) (Log Off Relational Database) statement, or another `@TRC` statement with no values in the `c,d,r` subfields.

---

## Syntax

```
@TRC[,c,d,r,db,...,db] .
```

> **Notes on cabinet/drawer fields:**
> - If cabinet and drawer are not specified, replace the field with one comma on OS 2200, or two commas on Windows Server, Windows Client, Linux, or UNIX systems.
> - *(OS 2200 only)* You must specify a full character set (FCS) drawer.

### Parameters

| Field | Platform | Description |
|-------|----------|-------------|
| `c,d,r` | All | Report to store the captured syntax. If omitted, terminates the previously issued `@TRC` statement for the specified database. |
| `db` | Windows / Linux / UNIX | Database name. Multiple database names may be specified. Default = default database name on your system. |
| `db` | OS 2200 | Database name. Multiple database names may be specified. Default = database running with `MRIMID` set to `1`. See your administrator for more information. |

Multiple database names can be combined in a single statement to start tracing one database while stopping another. For example, the following starts saving syntax sent to the Salary database and stops saving syntax to the Payroll database:

```
@trc,0,a,3,salary,,,payroll .
```

---

## Examples

Save SQL syntax to report `5A0` for the Oracle database:

```
@trc,0,a,5,oracle .
```

Terminate a previously issued `@TRC` statement for the Oracle database:

```
@trc,,,oracle .
```
