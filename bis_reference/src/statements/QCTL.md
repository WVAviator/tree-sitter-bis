# @QCTL (Queue Control)

*(2200 only)*

## Overview

Obtains various information about the Data Transfer Module (DTM).

---

## Syntax

```
@QCTL[,c,d,r,lab] func[,p] v[,...,v] .
```

### Parameters

| Field | Description |
|-------|-------------|
| `c,d,r` | DTM configuration report. For use by administrators. |
| `lab` | Label to go to in case of error. |
| `func` | `INFO` or `STAT`, depending on the kind of information needed. For additional administration functions, see the Administration Guide. |
| `p` | Parameters (up to 10). As an application designer, the only parameter you can specify is a queue name. All other parameters are specified by the administrator. |
| `v` | Variables to load. See [Functions for Loading Variables](#functions-for-loading-variables). |

---

## Reserved Words

| Word | Description |
|------|-------------|
| `STAT1$` | Error status corresponding to standard system messages. Use [`@LSM`](LSM.md) to read the message. |
| `STAT2$` | Line number in the DTM configuration report where the error occurred, if applicable. For use by administrators. |
| `STAT3$` | Field number in the DTM configuration report where the error occurred, if applicable. For use by administrators. |

If the system encounters an error condition, it returns an error status and stops processing. All actions completed before the error remain in place.

---

## Functions for Loading Variables

### INFO Function

Loads up to 13 variables with general information about the message. This information is defined only if the run is on a station processing input from DTM (`DTM$ = 2`).

| Variable | Type/Size | Content |
|----------|-----------|---------|
| Message Type | I1 | Message type code. See [Message Types](#message-types). |
| Processing Queue | H12 | Name of the input queue receiving the report currently in process. |
| Origin Queue | H12 | Name of the originating queue sending the report currently in process. |
| Character Set | I1 | Not used at this time (always blank). |
| Station Number | I5 | Station number initiating the original message, to which responses are directed, or the value specified in the `@QSND` or `@QSNR` statement. |
| Accounting Department Name | H12 | Name of the department where the originating user is signed on, or the value specified in the `@QSND` or `@QSNR` statement. |
| Status Code | H8 | Error code returned with message types 3 or 4. If type 4 (generated by the DTERR run), contains the 6-character system message mnemonic. All spaces for message types 0, 1, and 2. |
| Error Number | I6 | Error number associated with a system message. If not the DTERR run, a nonfatal error encountered in receiving the input report. If the DTERR run, the error that caused it to be called. |
| Name of Erring Run* | H12 | Name of the run that caused the error. |
| Name of Erring Function* | H8 | Name of the function within the run that caused the error, if a run error called the DTERR run. |
| RCR Drawer* | O6 | Octal drawer containing the run control report being interpreted at the time of error. |
| RCR Report* | I4 | Report number in the drawer of the run control report. |
| RCR Line Number* | I6 | Line number within the run control report that contains the error. |

*\* Variables used only by the DTERR run.*

#### Message Types

| Value | Description |
|-------|-------------|
| `0` | Generated by [`@QSND`](QSND.md) (Send Message, No Response). No response is expected. |
| `1` | Generated by [`@QSNR`](QSNR.md) (Send Message, Response Expected). The processing run must generate a response; otherwise an error occurs. |
| `2` | Generated by [`@QRSP`](QRSP.md) (Send Response Message). A normal response to a `@QSNR` message sent by this system. |
| `3` | An abnormal response to a `@QSNR` message sent by this system; otherwise the same as type 2. See the `err` field under `@QRSP` for status. |
| `4` | Generated by the DTERR run in response to a `@QSNR` message. An error response indicating the message text is in a specific format. Could represent several different errors (e.g., the run at the destination failed, could not be started, or sign-on data was invalid). |
| `5` | An error occurred at the destination and all normal error handling failed. No significant information is returned — the problem must be solved at the destination. This is a courtesy response to a waiting run; a valid delayed response may still arrive. |
| `6` | The message headers did not contain a valid message type. A fatal input error used only by the DTERR run. |
| `7` | The message is closed out by a [`@QRSP`](QRSP.md) or [`@QREL`](QREL.md) (Release Message) statement. Neither `@QRSP` nor `@QREL` should be used again in this run. |

### STAT Function

Loads up to 15 variables with status information about the queue specified in the parameter field. Exactly one queue must be specified. Can be used at any terminal as long as the system is configured with DTM.

| Variable | Type/Size | Content |
|----------|-----------|---------|
| Status Flags | H8 | Eight Y/N flags examined as a group, in order: send-enabled, receive-enabled, configured for input, initialized for input, not manually suspended, not suspended due to error, not suspended for configuration change, configured correctly. Examples: normal input queue = `YYYYNNNN`; normal output queue = `YYNNNNNN`. |
| Queue Status from DTM | I1 | Status of the named queue within DTM common banks. `0` = does not exist, `1` = exists but inactive, `2` = active but in slowdown, `3` = active. Requesting this variable may involve one or more I/O requests. |
| Queue Count | I5 | Number of unprocessed reports currently on the queue. |
| Queue Maximum | I5 | Maximum number of reports on the queue. |
| Input Priority | I1 | Input priority of the queue from the DTM configuration report. Zero if not configured for input. |
| Max Activities | I1 | Maximum number of active input reports that can be processed simultaneously. |
| Current Activities | I1 | Number of reports currently being processed. |
| Input Count | I6 | Number of reports processed on the queue since last initialization. |
| Output Count | I6 | Number of reports generated by the queue since last initialization. |
| To Count | I6 | Number of reports sent to the queue since last initialization. |
| From Count | I6 | Number of reports received from the queue since last initialization. |
| Direct Response Count | I6 | Number of responses received while a run was waiting for the response. |
| Indirect Response Count | I6 | Number of responses received without a run waiting. Not necessarily an indication of timeout or other error. |
| Input Error Count | I4 | Number of input messages directed to the DTERR run for reasons other than process errors (e.g., bad sign-on information, badly formatted input, unknown run name). |
| Process Error Count | I4 | Number of input messages directed to the DTERR run because of an error in the processing run. |
| Time of Last Activity | H8 | Time the last activity occurred. |

## Example

Use the INFO function to request general information about a message:

```
@qctl info <mtyp>i1,,<oque>h12,,<sn>i5 .
```

| Variable | Description |
|----------|-------------|
| `<mtyp>i1` | Load `<mtyp>` with the message type. |
| `<oque>h12` | Load `<oque>` with the name of the originator queue. |
| `<sn>i5` | Load `<sn>` with the unit number of the originator. |

Examine the message type and send a response if necessary (using the current result as the response report):

```
@if <mtyp> eq 1 qrsp,-0 ; .
```
