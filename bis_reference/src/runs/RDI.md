# RDI (Relational Database Interface)

## Overview

Provides an interactive interface between the system and several relational database management systems. RDI enables users to easily access data stored in relational databases by responding to prompts on menus, eliminating the need to formulate complete SQL statements manually.

> **Note:** One or more databases must be registered before use. See your administrator for more information.

### RDI and SQL

RDI enables you to submit standard SQL statements in a familiar, menu-driven format. SQL can be used to create and alter tables, retrieve data from single or multiple tables, and insert, change, or delete relational data.

To save an SQL command generated by RDI to a report:

1. Select the desired SQL task from the RDI menu and respond to the prompts on the assistance screen.
2. RDI translates the entered information into an SQL command and submits it to the database.
3. The selected data is returned as a report, or the relational operation is performed.
4. Press **ShwSQL** to view the generated SQL statement.
5. Press **SveSQL** to save the SQL statement to a report.

---

## Syntax

```
RDI[,command,drawer,db,user-id,psw]
```

### Parameters

| Field | Description |
|-------|-------------|
| `command` | Displays a command-specific menu. See [Commands](#commands) and [Utilities](#utilities) for valid values. |
| `drawer` | Drawer to contain the result. |
| `db` | Database name, if different from the default. |
| `user-id` | User-ID for database access, if different from the default. |
| `psw` | Password for database access. Required only for some database managers. |

---

## Commands

| Command | Description |
|---------|-------------|
| `select` | Retrieves rows from a single relational table into a report. |
| `join` | Retrieves rows from two or more relational tables into a report. |
| `insert` | Adds new rows to a relational table. |
| `delete` | Deletes rows from a relational table. |
| `update` | Updates relational tables with requested changes. |
| `create` | Defines new tables in the database. |
| `sql` | Submits SQL syntax directly to the database. SQL syntax generated by RDI can also be saved for later use. |

---

## Utilities

| Utility | Call | Description |
|---------|------|-------------|
| Alter table | `alter` | Changes an existing table as specified. |
| Restrict table | `restrict` | Controls access to tables and views by user-ID and RDI command. Enables the table owner to grant, revoke, and restore access. |
| Destroy table | `destroy` | Permanently removes a table definition and all its data rows from the database. |
| Create view | `cview` | Defines new views in the database. |
| Drop view | `dview` | Removes existing views from the database. |

---

## Guidelines

- To select a command from the RDI menu, tab to the desired selection and transmit.
- RDI menu commands not accessible with your user-ID are displayed in blue on a color monitor, or in low intensity on some black-and-white monitors. Contact your administrator for information about user-ID privileges.
- If a database requires a password, the DBMS menu is displayed. To accept the default database and user-ID, enter a password in the appropriate field if required.
- To display the DBMS menu manually and select a database, press **DBMS** from the RDI main menu.
- To change the default parameters used during RDI operations, press **Envmnt** and enter new information.
- *(RDMS 2200 only)* The RDI session will be read-only when a user is configured for multiple logons = `Y` and RDI is invoked with `SELECT` or `JOIN` as the first command (either explicitly or as the first action taken). This applies only to RDMS 2200 database models.
