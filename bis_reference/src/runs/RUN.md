# RUN — Automatic Run Generation

## Overview

Use the Automatic Run Generation (`RUN`) run to generate statements automatically from functions you perform at your terminal. Run generates these statements transparently as you execute the functions.

Run is especially useful for users who perform a repeated sequence of tasks on the same reports. A user with no knowledge of run design can easily automate basic tasks using Run.

For example, if you need to search a report for a particular item, sort the result, add the values in a column, and print the result every day, you can have Run generate statements for each of these tasks — then all you need to type is one word: the run name.

When started, your terminal is placed under control of Run. If no report is on display, the Automatic Run Generation active screen is displayed; otherwise the run redisplays the report.

---

## Syntax

```
RUN[*]
```

| Option | Description |
|--------|-------------|
| `*` | Creates statements using field names. Default (no `*`) creates statements using column-character positions. |

---

## Procedures

After starting the `RUN` command, follow these steps:

1. Perform the commands for which you want statements generated.
2. Use the keys on the [Run Function Key Bar](#run-function-key-bar) to control the operation of the Run command.
3. Press **Exit** or type a caret (`^`) and transmit when you are ready to register the statements as a run or to execute them.

To register your statements as a run, in the **Automatic Run Generation Terminated** screen, type the letter of the drawer where you want the run to reside and transmit.

> **Note:** You may be required to register the run in a drawer designated for run control reports (with a drawer name beginning with "Run"). Your administrator can change this to allow registration in any kind of drawer.

When you register, the Run command:

- Duplicates the result containing the statements in the drawer you specify in your current cabinet.
- Registers the run with a run name matching your user-id, restricted so that only your user-id can execute it.
- Overwrites any previously generated run with the same run name. To retain the old run, have the administrator reregister it with a new name before running again.

> *(2200 only)* If your user-id begins with a number (e.g., `7NEWUSER`), you can still execute the run by entering `run 7newuser`.

---

## Displaying Reports and Results in Your Generated Run

The Run command does not generate a [`@DSP`](../statements/DSP.md) statement for each result generated by a command, even though results are visible as you execute commands. It generates a `@DSP` statement only after the final result in the run.

To display each intermediate result, use the manual **Display Report** (`D`) function. For example, to display both a search result and a sort result, type `d -` after each result is displayed and transmit.

---

## Limitations

- The following commands can be used while under control of Run, but Run **will not** generate statements for them:
  - `L` — restores the control line (same as typing `pnt` and transmitting).
  - `RSM` — resumes the run (same as pressing **DspRun**).
- Run only generates statements for commands that have equivalent manual functions. For example, the [`@LDV`](../statements/LDV.md) (Load Variable) statement has no equivalent manual function and cannot be generated.
- The following commands generate statements but **terminate** automatic run generation:
  - `X` — generates an [`@XIT`](../statements/XIT.md) (Sign Off User Session) statement and displays the Automatic Run Generation Terminated screen.
  - `^` — generates a [`@REL`](../statements/REL.md) (Release Display) statement. The run executes, releases the screen, and displays the Automatic Run Generation Terminated screen.
- You cannot generate output screens or capture user input from any other source (e.g., `INPUT$` or `INVAR$`).
- You cannot reuse a function mask after a system message is displayed on the control line — you must reenter the call, complete the function mask a second time, and transmit.
- You cannot use the SOE Update command during run generation. However, when the generated run is executing, you can update reports normally using the Change command, an updating command such as Search Update followed by Update, or a line change command such as Add Line.

## Run Function Key Bar

| Key | Description |
|-----|-------------|
| **DspRun** | Displays the generated statements in a result. |
| **Re-Dsp** | Displays the report you were processing. Use after pressing **DspRun** to continue executing other commands after reviewing the generated statements. |
| **AcScrn** | Displays the Automatic Run Generation active screen. |
| **Exit** | Displays the Automatic Run Generation Terminated screen, where you can register the run or use the following function keys: **DspRun** (displays generated statements in a result) or **Run** (executes the statements — press **Resume** for each `@DSP` statement to continue processing). |
| **Quit** | Terminates the Run command and displays the active screen. |
