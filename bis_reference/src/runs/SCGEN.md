# SCGEN — Screen Control Generator

## Overview

Use the `SCGEN` command to create input screens and menus using a menu-driven interface. You design screens using a visual layout that resembles the actual output screen, and can include fields already defined in the data dictionary. `SCGEN` automatically generates the [`@SC`](../statements/SC.md) (Screen Control) code for use in your run.

When the screen design is final and you press **Build**, the screen control code is created and placed into a report in drawer A of cabinet 0. Your run can then use the [`@SC`](../statements/SC.md) statement to access that report.

---

## Syntax

```
SCGEN
```

---

## Guidelines

### Creating Screen Text

Place text anywhere on the scratch pad, except directly over existing fields or on frame borders. Move the cursor to where you want the text to begin and start typing. The command automatically updates text on the screen whenever you transmit or select a new option.

> **Caution:** Do not use underscores (`_`), quotation marks (`"`), or vertical bars (`|`) in screen text — these characters can cause translation problems.

You can use reserved words in screen text. To display the contents of a reserved word, type it normally (for example, `TIME$`).

### Changing the Display to Black on White

To override the color attributes of menus generated by `SCGEN`, set the `BlackOnWhite` parameter in the `MPC.INI` file to `true` (1). See the *Graphical Interface for Business Information Server* for more information on the `MPC.INI` file.
